1. Visão geral

O OpenCircuit é um editor/simulador visual didático de lógica digital. Ele suporta circuitos combinacionais, blocos combinacionais nativos, componentes sequenciais nativos e memória emergente por realimentação usando portas comuns.

Marco atual: a base foi refatorada de arquivos monolíticos para uma arquitetura modular. O App.tsx e o CircuitCanvas.tsx estão muito menores, e o core tem módulos claros em src/core/simulation/.

O projeto continua intencionalmente focado em lógica digital. Não é simulador elétrico analógico.

3. Modelo de dados

interface CircuitDocument {
  version: 1;
  components: LogicComponent[];
  wires: Wire[];
}

interface LogicComponent {
  id: string;
  type: GateType;
  x: number;
  y: number;
  label?: string;
  state?: boolean;
  width?: number;
  memory?: Record<string, boolean>;
}

O documento continua simples: componentes e fios. O campo memory é usado por componentes sequenciais nativos, como d-latch e d-flip-flop.

Já a memória emergente de circuitos com feedback fica no runtime de simulação, não no JSON do circuito.

7. Testes

Comando principal:

npm test

Hoje os testes cobrem:

  • XOR combinacional básico;
  • Latch SR com NOR mantendo estado;
  • Latch SR com NAND ativo baixo mantendo estado;
  • Latch D construído com NANDs;
  • NOT realimentado detectado como instável;
  • detecção de feedback no grafo;
  • stepCircuit() com borda de subida/descida;
  • tabela verdade do XOR;
  • roteamento de self-loop/balão.

Validação completa recomendada:

npm test
npm run build

8. Fluxos importantes

Simulação normal

const { simulationResult, evaluation, resetSimulationRuntime } = useSimulationRuntime(circuit);

O runtime preserva sinais anteriores para feedback. Use resetSimulationRuntime() para limpar memória emergente sem apagar o circuito.

Tick sequencial

setCircuit((current) => stepCircuit(current));

Alterna clocks e atualiza memória de latches/flip-flops nativos.

Tabela verdade

É exibida somente quando não há sequenciais nem feedback. Caso contrário, o painel vira estado do circuito.

9. Como adicionar componentes

  1. Adicionar o tipo em GateType, em src/core/types.ts.
  2. Adicionar definição visual/lógica em src/core/catalog.ts.
  3. Adicionar avaliação em src/core/simulation/gates.ts ou comportamento sequencial em sequential.ts.
  4. Adicionar na biblioteca em src/ui/library/ComponentLibrary.tsx.
  5. Adicionar prefixo de ID em src/ui/app/editorUtils.ts.
  6. Adicionar exemplo didático em src/examples/circuitExamples.ts, se fizer sentido.
  7. Adicionar teste em tests/core/simulation.test.ts.

10. Cuidados ao manter

  • Não quebre a compatibilidade do JSON version: 1.
  • Não confunda estado do documento com estado runtime da simulação.
  • Ao mexer em feedback, teste latch NOR, latch NAND e NOT self-loop.
  • Ao mexer em canvas, teste drag, seleção múltipla, conexão por clique, conexão por drag, self-loop, pan e zoom.
  • Ao mexer em histórico, evite registrar estados demais durante drag/resize.
  • Ao mexer em tabela verdade, garanta que a avaliação temporária não muta o circuito real.
Atenção: os arquivos estão mais modulares, mas ainda há interações complexas entre canvas, seleção, fios e histórico. Faça mudanças pequenas e rode testes sempre.
