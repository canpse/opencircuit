1. Visão geral do produto

O OpenCircuit é um editor visual didático de circuitos lógicos digitais, inspirado em ferramentas como Logisim, mas com uma UI moderna, simples e fluida.

Escopo atual: lógica digital combinacional, simulação instantânea, tabela verdade automática, exemplos explicados e blocos combinacionais nativos.

O projeto não é um simulador elétrico analógico/DC e não tenta modelar tensão, corrente ou componentes físicos.

2. Arquitetura e arquivos principais

src/core/types.ts Tipos centrais: circuito, componente, fio, pino e resultado de avaliação.
src/core/catalog.ts Catálogo de componentes: tamanho, label e posições dos pinos.
src/core/evaluateCircuit.ts Motor de simulação combinacional.
src/ui/App.tsx Orquestra estado global, histórico, exemplos, import/export, tabela verdade e biblioteca.
src/ui/editor/CircuitCanvas.tsx Canvas SVG, renderização de componentes/fios e interações de edição.
src/ui/styles.css Visual geral da aplicação.
src/state/storage.ts Autosave, carregamento inicial e exportação JSON.

3. Modelo de dados

Um circuito é um documento JSON simples:

interface CircuitDocument {
  version: 1;
  components: LogicComponent[];
  wires: Wire[];
}

Um componente possui tipo, posição e metadados opcionais:

interface LogicComponent {
  id: string;
  type: GateType;
  x: number;
  y: number;
  label?: string;
  state?: boolean;
  width?: number; // usado principalmente por blocos de texto
}

Um fio conecta um pino de saída a um pino de entrada:

interface Wire {
  id: string;
  from: { componentId: string; pinId: string };
  to: { componentId: string; pinId: string };
}
O caminho visual do fio não é salvo. Ele é calculado automaticamente no canvas a partir de from e to.

4. Catálogo de componentes

Todo tipo de componente precisa estar declarado em src/core/catalog.ts. Ali ficam largura, altura e pinos.

5. Simulação

A simulação fica em src/core/evaluateCircuit.ts. Ela é iterativa e combinacional, com limite de segurança. 

const MAX_ITERATIONS = 64;

Para componentes com lógica própria, adicione um case em evaluateComponent(). 

case 'half-adder': {
  const a = inputValue(circuit, values, componentById, component.id, 'A');
  const b = inputValue(circuit, values, componentById, component.id, 'B');
  return writeMany(values, component.id, {
    SUM: a !== b,
    CARRY: a && b,
  });
}
Circuitos sequenciais, clock, flip-flops e memória de estado exigirão outra abordagem de simulação.

6. Canvas SVG e interações

O editor visual é SVG. O arquivo principal é src/ui/editor/CircuitCanvas.tsx.

  • Fios são <path>.

  • Zoom/pan usam alteração de viewBox.
  • Texto editável usa estado local e foreignObject para input HTML.

7. Fios

Existem dois estilos visuais, selecionáveis na barra superior:

  • Ortogonal: segmentos retos, tenta evitar componentes e desenha “pulos” em cruzamentos.
  • Curvo/Bézier: visual mais suave, parecido com o estilo inicial.

A preferência é salva em localStorage, não no circuito.

O roteamento ortogonal é visual/calculado. Se o circuito for exportado, apenas origem e destino dos fios são salvos.

8. Histórico, undo e redo

O histórico é manual em App.tsx. Antes de mudanças estruturais, chame rememberCircuit().

  • Adicionar/remover componente: registra histórico.
  • Renomear: registra histórico.
  • Carregar exemplo/importar: registra histórico.
  • Drag/resize contínuo: deve registrar apenas uma vez no início da ação.
function renameComponent(componentId: string, label: string) {
  const currentComponent = circuit.components.find(c => c.id === componentId);
  if (!currentComponent || currentComponent.label === label) return;
  rememberCircuit();
  // altera estado...
}

9. Tabela verdade

A tabela verdade é do circuito inteiro por padrão:

  • Entradas: todos os componentes input.
  • Saídas: todos os componentes led.
  • Gera combinações até um limite.
  • Avalia cópias temporárias do circuito, sem alterar o circuito real.
  • Destaca a linha correspondente ao estado atual dos inputs.
const testCircuit: CircuitDocument = {
  ...circuit,
  components: circuit.components.map(component =>
    component.type === 'input'
      ? { ...component, state: inputValueById.get(component.id) ?? false }
      : component
  ),
};

11. Como adicionar um novo componente lógico

  1. Adicionar o tipo em GateType, no arquivo src/core/types.ts.
  2. Adicionar definição em COMPONENT_DEFINITIONS, no catalog.ts.
  3. Implementar avaliação em evaluateCircuit.ts.
  4. Adicionar na biblioteca em TOOL_GROUPS, no App.tsx.
  5. Atualizar prefixo de ID em nextId().
  6. Rodar npm run build.
Se o componente só for visual, como text, ele pode ter pins: [] e retornar false na avaliação.

12. Cuidados ao manter/extender

  • Verifique se pinos do catálogo batem com os nomes usados na simulação.
  • Não mude o formato JSON sem pensar em compatibilidade.
  • Não salve caminhos visuais de fios sem uma decisão explícita de produto.
  • Ao mexer em interações SVG, teste clique, drag, menu de contexto e seleção múltipla.
  • Ao mexer em texto editável, teste Enter, Esc, blur e undo.
  • Ao mexer em tabela verdade, garanta que ela não muta o circuito real.
  • Ao mexer em histórico, evite registrar centenas de estados durante drag contínuo.
  • Sempre rode npm run build.

13. Próximos caminhos possíveis

Consolidar combinacional

  • Copiar/exportar tabela verdade como CSV/Markdown.
  • Melhorar auto-layout de fios.
  • Adicionar mais exemplos didáticos.

Subcircuitos

Permitir que o usuário crie um circuito e transforme em bloco reutilizável. É poderoso, mas muda bastante o modelo.

Lógica sequencial

Clock, latch, flip-flop, registradores e contadores. Esse é o próximo grande capítulo, mas requer simulação por ticks/estado.
